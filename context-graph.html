Technical lineage

From raw ingest to executive dashboard — one traceable path

The context graph ingests metadata from Fivetran syncs, dbt manifests, Airflow DAGs, warehouse catalogs, and BI tools. Every edge is grounded in synced metadata — not LLM guesses. Click any node and traverse upstream sources or downstream consumers in seconds.

Impact analysis runs on this graph: rename a column in stg_orders and see every model, DAG task, and dashboard that breaks before you merge.

  • Cross-system lineage without manual diagram maintenance
  • Column-level dependencies where metadata supports it
  • Freshness and ownership propagated to every node
  • Export for SOC2, GDPR, and examiner requests
Business context layer

Metrics, glossaries, and policies on the same nodes

Technical assets do not exist in isolation. The context graph attaches certified metric definitions, business glossary terms, data classifications, and ownership to warehouse tables, dbt models, and BI explores — so "MRR" is not just a name but a node with lineage, owners, and reconciliation history.

Finance, product, and engineering query the same graph from different angles. No parallel wiki, no spreadsheet of definitions that diverged in Q2.

  • Canonical KPIs linked to upstream models and source tables
  • Business glossary terms mapped to technical assets
  • PII and classification tags with downstream consumer lists
  • Reconciliation events when definitions change
