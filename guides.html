Infrastructure

How Metroflow understands your stack

Metadata crawl → semantic graph → query lineage and impact. No data movement — just the context your team already wishes lived in one place.

Step 1

Connect sources

Start by pointing Metroflow at the systems you already run. Snowflake, BigQuery, dbt Cloud, Airflow, Dagster, Fivetran, and 40+ connectors ship out of the box — with more added regularly in the open-source repo.

Each connection uses read-only metadata crawls. Metroflow never copies row-level data to its servers; it ingests manifests, schemas, DAG definitions, and job histories only.

Credentials stay in your network. For self-hosted deployments, secrets live in your vault or environment — not a multi-tenant SaaS database.

  • Pick connectors from the integrations catalog — OAuth or key-based auth
  • Scope crawls per environment: prod, staging, and dev stay separated
  • Schedule syncs from hourly to daily — or trigger on deploy webhooks
  • Health dashboard shows last sync, errors, and assets discovered per source
  • Under 5 minutes from docker compose up to first connector live
Step 2

Build the semantic graph

Raw metadata from each connector is normalized into a unified graph. dbt model fct_revenue links to its Snowflake table, upstream Airflow task, and downstream Looker explores — automatically.

The graph stores ownership, tags, freshness, column types, and test results alongside relationships. Search and AI agents query this layer — not individual siloed APIs.

Incremental syncs update only what changed. Large warehouses with thousands of tables stay manageable because Metroflow diffs each crawl.

  • Manifest parsing for dbt Core, Cloud, and CI artifact uploads
  • DAG-to-model mapping for Airflow, Dagster, and Prefect
  • Column-level lineage merged where connectors expose it
  • Automatic owner inference from repo CODEOWNERS and warehouse tags
  • Graph API for custom tools, agents, and internal portals
Step 3

Query & act

Once the graph is live, your team queries lineage, runs impact analysis, and works in stack-aware editors — all grounded in the same metadata.

Rename a column and preview blast radius before opening a PR. Ask Enterprise Brain why a dashboard is stale and get a trace to the Airflow task that slipped.

Infrastructure understanding becomes daily workflow — not a quarterly lineage initiative that rots in a wiki.

  • Interactive lineage: click any node, walk upstream or downstream
  • Impact reports exportable for architecture reviews and change management
  • Editors surface refs, schemas, and warnings inline as you type
  • Agents answer stack questions with citations to graph nodes
  • RBAC ensures analysts see only the environments they're cleared for

Metrics

How canonical metrics work

From definition to trusted KPI — with reconciliation and full provenance on the same graph that powers your engineering workflows.

Define

Certify on the semantic layer

Create metric definitions that reference your dbt semantic models or warehouse tables directly. Grain, filters, dimensions, and business logic live in one object — not scattered across BI folders.

Link each KPI to stewards in finance and data. Draft definitions go through review before the certified badge appears — preventing "quick fixes" from becoming exec-facing truth.

Because metrics sit on the infrastructure graph, every definition inherits lineage to source columns automatically.

  • Start from templates: MRR, churn, DAU, conversion — customize filters
  • Attach to dbt metrics, semantic layer, or raw SQL with governance wrapper
  • Plain-English description field for non-technical stakeholders
  • Version history with approver sign-off on every publish
  • Preview lineage before certifying — see upstream models and PII exposure
Standardize

Push to every tool

Certified metrics sync to the surfaces your teams already use. BI explores, notebooks, internal APIs, and Metroflow agents all resolve to the same canonical ID.

When a definition updates, consumers get notified — not surprised in a Monday leadership meeting. Override detection flags dashboards still running rogue SQL.

Standardization is push, not policy PDFs. Make the right number the easiest number to use.

  • API and SDK for embedding certified metrics in custom apps
  • Looker, Tableau, Metabase, and Hex integration guides in docs
  • Agents auto-resolve ambiguous terms to certified definitions
  • Consumption analytics — which teams use which KPIs
  • Environment gates — staging metrics never appear in prod dashboards
Reconcile

Audit when definitions change

Trust requires verification. Schedule reconciliation jobs that compare certified metrics against billing systems, CRMs, and ERP exports — surfacing gaps with suggested causes.

When definitions change, exportable audit trails capture the diff, approver, and affected dashboards. Finance close and governance reviews get evidence — not screenshots.

Enterprise Brain ties reconciliation results to infrastructure events — "gap explained by 6h pipeline delay in stg_orders."

  • Scheduled Stripe, Salesforce, and warehouse cross-checks
  • Alert when certified vs. source-of-record delta exceeds threshold
  • PDF/CSV audit export with lineage, owners, and change log
  • Board prep mode: top KPIs with reconciliation status badges
  • Webhook notifications to Slack when metrics need steward review
