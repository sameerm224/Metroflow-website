Enterprise Brain

Technical + business reasoning in one layer

Enterprise Brain is Metroflow's unified reasoning layer — it combines infrastructure lineage with canonical metric definitions so you can ask questions that span both worlds.

When churn spikes, Brain doesn't stop at "here's a SQL query." It checks whether the certified churn metric changed, whether stg_subscriptions was delayed, and which deploy coincided with the move — then summarizes with citations.

One agent for engineers debugging pipelines and executives preparing board decks. Same ground truth, different depth.

  • "Why did churn increase last week?" — metric change + pipeline delay + deploy log
  • "Which models feed board-level revenue?" — lineage to certified MRR definition
  • "Reconcile MRR between finance and product dashboards" — diff uncertified SQL
  • Cited sources in every answer: models, metrics, owners, timestamps
  • Conversation memory scoped to your workspace — not a public model's training set

Natural language queries

Ask in plain English over stack & metrics

Not everyone writes SQL — but everyone has questions about the stack. Natural language queries let analysts, PMs, and leaders ask about lineage, impact, and certified KPIs in plain English.

Metroflow resolves entities against your catalog ("revenue" → certified monthly_recurring_revenue) before generating an answer. Ambiguous terms prompt clarification instead of silent wrong assumptions.

Every response links back to the models, tables, and metric definitions used — so recipients can verify, not just trust.

  • "Show me everything upstream of the board deck" — subgraph + freshness
  • "Why did NRR drop last week?" — Brain traces pipeline + definition changes
  • "Who owns customer data models?" — owners, Slack contacts, recent deploys
  • Export answers as shareable briefs with embedded lineage screenshots
